Desktop visitors get the full article, while mobile sessions receive a paywall preview.

With the gvm.js hide (mobile only) strategy, you can easily wrap your existing website. During page load, our scripts hide some content and display a paywall. Once the user interacts with the paywall, the content becomes visible after a successful payment. This is the simplest way to integrate GetViaMsg—it’s not perfect, but it’s easy and functional.

There are some security considerations—for instance, advanced users could bypass this integration by disabling JavaScript or inspecting network traffic. However, this approach is still valuable for testing engagement and understanding how many users interact with the payment solution.

On mobile devices, bypassing this integration is more difficult. Globally, 90% of digital content traffic comes from mobile devices, making this strategy practical.

Conditional integration is also supported. You can enable gvm.js only for specific devices, traffic sources, or social media campaigns.

The hide strategy includes additional templates for locking content and the payment modal. Check the source code in this file to see how everything is declared and connected.
